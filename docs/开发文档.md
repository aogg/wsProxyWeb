# WebSocket代理网页请求系统 - 开发文档

## 项目概述

本项目实现一个通过浏览器插件和WebSocket代理网页请求的系统。包含两个独立的子项目：

- **server/**：Go语言服务端
- **client/**：TypeScript浏览器插件客户端

## 通信协议

详见 [协议说明.md](./协议说明.md)

### 消息类型

| 类型 | 方向 | 说明 |
|------|------|------|
| ping | 客户端 → 服务端 | 心跳请求 |
| pong | 服务端 → 客户端 | 心跳响应 |
| request | 客户端 → 服务端 | HTTP请求代理 |
| response | 服务端 → 客户端 | HTTP响应返回 |
| error | 服务端 → 客户端 | 错误消息 |
| close | 双向 | 关闭连接 |

### 消息流程

```
1. 客户端拦截网页请求
2. 封装为request消息 → 压缩 → 加密 → 发送
3. 服务端接收 → 解密 → 解压 → 解析
4. 服务端执行HTTP请求
5. 服务端封装响应 → 压缩 → 加密 → 发送
6. 客户端接收 → 解密 → 解压 → 解析
7. 客户端返回响应给网页
```

## 常见开发问题

### 1. Service Worker休眠

Manifest V3的Service Worker会在空闲时休眠，导致状态丢失。

**解决方案**：
- 使用`chrome.storage`持久化状态
- 通过心跳保持连接
- 监听`chrome.runtime.onInstalled`事件

### 2. 请求拦截权限

需要在manifest.json中声明正确的权限：

```json
{
  "permissions": [
    "webRequest",
    "webRequestAuthProvider",
    "storage",
    "tabs"
  ],
  "host_permissions": [
    "<all_urls>"
  ]
}
```

### 3. 跨域问题

服务端执行请求时不受浏览器跨域限制，但需要正确处理CORS头。

### 4. 加密密钥管理

- 密钥必须是32字节
- 使用Base64编码存储
- 生产环境建议使用密钥管理服务

# WebSocket代理网页请求系统 - 使用说明

## 项目简介

本项目实现一个通过浏览器插件和WebSocket代理网页请求的系统，支持Chrome和Edge浏览器。可以将网页请求通过WebSocket转发到代理服务器执行，实现请求代理、数据加密传输等功能。

## 系统要求

### 服务端
- Go 1.24.0 或更高版本
- 支持的操作系统：Windows、Linux、macOS

### 客户端（浏览器插件）
- Node.js 20.0 或更高版本
- TypeScript 5.9 或更高版本
- Google Chrome 88+ 或 Microsoft Edge 88+
- 需要支持 Manifest V3

## 快速开始

### 1. 安装依赖（首次运行）

```powershell
# 客户端依赖
cd client && npm install && cd ..

# 服务端依赖
cd server && go mod download && cd ..
```

### 2. 构建项目

```powershell
# 构建客户端和服务端
.\bin\build.bat
```

### 3. 启动服务端

```powershell
.\bin\start-server.bat          # 使用默认端口 8080
.\bin\start-server.bat 9090     # 指定端口
```

服务端将在 `ws://localhost:8080/ws` 监听WebSocket连接。

### 4. 安装浏览器插件

构建完成后，插件文件位于 `client/dist` 目录。

#### 在Chrome中加载插件

1. 打开Chrome浏览器，访问 `chrome://extensions/`
2. 开启右上角的「开发者模式」
3. 点击「加载已解压的扩展程序」
4. 选择 `client/dist` 目录
5. 插件安装成功后，工具栏会显示插件图标

#### 在Edge中加载插件

1. 打开Edge浏览器，访问 `edge://extensions/`
2. 开启左下角的「开发人员模式」
3. 点击「加载解压缩的扩展」
4. 选择 `client/dist` 目录
5. 插件安装成功后，工具栏会显示插件图标

## 配置说明

### 服务端配置

配置文件位于 `server/src/configs/server.yaml`：

```yaml
# 服务器配置
server:
  port: "8080"  # WebSocket服务器监听端口

# 加密配置
crypto:
  enabled: false  # 是否启用加密
  key: ""  # 加密密钥（32字节，Base64编码）
  algorithm: "aes256gcm"  # 加密算法：aes256gcm 或 chacha20poly1305

# 压缩配置
compress:
  enabled: false  # 是否启用压缩
  level: 6  # 压缩级别：1-9
  algorithm: "gzip"  # 压缩算法：gzip 或 snappy

# 安全控制配置
security:
  enabled: true
  maxConnections: 50  # 最大连接数
  maxMessageBytes: 2097152  # 单条消息最大字节数
  maxRequestBodyBytes: 5242880  # 请求体最大字节数
  rateLimitPerSecond: 50  # 每秒允许的请求数
  allowIPs: []  # 允许IP列表
  denyIPs: []  # 拒绝IP列表
  allowDomains: []  # 允许域名列表
  denyDomains: []  # 拒绝域名列表

# HTTP客户端配置
http:
  timeoutSeconds: 30  # 请求超时秒数
  maxIdleConns: 200  # 全局最大空闲连接数
  maxIdleConnsPerHost: 50  # 单host最大空闲连接数
```

### 客户端配置（插件弹窗）

点击插件图标打开配置界面，所有配置都在插件中完成：

#### 服务端连接配置
- **WebSocket地址**：设置代理服务器的WebSocket地址（默认：`ws://localhost:8080/ws`）

#### 加密设置
- **启用加密**：开关控制是否启用数据加密
- **加密密钥**：32字节密钥（Base64编码）
- **加密算法**：AES-256-GCM 或 ChaCha20-Poly1305

#### 压缩设置
- **启用压缩**：开关控制是否启用数据压缩
- **压缩级别**：1-9（级别越高压缩率越高，速度越慢）
- **压缩算法**：Gzip 或 Deflate

#### 拦截规则
- **启用请求拦截**：开关控制是否拦截网页请求
- **域名白名单**：只代理指定域名的请求（每行一个）
- **域名黑名单**：禁止代理指定域名的请求（每行一个）
- **URL模式**：使用通配符匹配URL（每行一个）

## 功能特性

### 1. 请求代理

- 自动拦截网页HTTP/HTTPS请求
- 通过WebSocket转发到代理服务器
- 支持GET、POST、PUT、DELETE等常见HTTP方法
- 支持请求头、请求体的完整转发

### 2. 数据加密

- 支持AES-256-GCM和ChaCha20-Poly1305加密算法
- 密钥可配置，支持Base64编码的32字节密钥
- 加密传输保障数据安全

### 3. 数据压缩

- 支持gzip和snappy压缩算法
- 可配置压缩级别（1-9）
- 减少传输数据量，提升性能

### 4. 安全控制

- 连接数限制
- 请求频率限制
- IP/域名黑白名单
- 请求大小限制

### 5. 性能优化

- HTTP连接池复用
- 并发请求处理
- 大文件分块传输

## 请求过滤规则

### 规则类型

1. **域名白名单**：只代理指定域名的请求
2. **域名黑名单**：禁止代理指定域名的请求
3. **URL模式匹配**：使用通配符匹配URL

### 规则语法

```
# 精确匹配
example.com

# 通配符匹配
*.example.com

# 路径匹配
example.com/api/*
```

### 配置示例

在插件弹窗中配置过滤规则：

```json
{
  "mode": "whitelist",
  "rules": [
    "*.api.example.com",
    "cdn.example.com/*"
  ]
}
```

## 常见问题

### 1. 插件无法连接服务器

**检查项**：
- 确认服务端已启动
- 检查服务器地址配置是否正确
- 检查防火墙是否开放端口
- 查看浏览器控制台是否有错误信息

### 2. 请求代理失败

**检查项**：
- 检查服务端日志
- 确认目标服务器可访问
- 检查请求是否被安全规则拦截
- 查看加密配置是否匹配

### 3. 加密通信失败

**解决方法**：
- 确保客户端和服务端的加密配置一致
- 检查密钥是否正确（32字节，Base64编码）
- 确认使用的加密算法相同

### 4. 性能问题

**优化建议**：
- 调整压缩级别（降低级别可提升速度）
- 优化过滤规则减少不必要的代理
- 检查网络连接质量
